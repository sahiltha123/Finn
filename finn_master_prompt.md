## FORM VALIDATION STRATEGY

### Domain Layer — Pure Dart Validators
```dart
// lib/core/utils/validators/amount_validator.dart
class AmountValidator {
  static String? validate(String? value) {
    if (value == null || value.isEmpty) return 'Amount is required';
    final amount = double.tryParse(value.replaceAll(',', ''));
    if (amount == null) return 'Enter a valid number';
    if (amount <= 0) return 'Amount must be greater than zero';
    if (amount > 10000000) return 'Amount exceeds maximum limit';
    return null;
  }
}
```

### Presentation Layer — flutter_form_builder Wiring
```dart
// In add_edit_transaction_sheet.dart
FormBuilderTextField(
  name: 'amount',
  keyboardType: const TextInputType.numberWithOptions(decimal: true),
  validator: FormBuilderValidators.compose([
    FormBuilderValidators.required(),
    (val) => AmountValidator.validate(val),  // Domain validator
  ]),
)
```

This pattern keeps business rules in domain (testable, reusable) while
flutter_form_builder handles form state, focus, and error display in the UI.

## GOALS SYSTEM — "FINN CHALLENGES" SPEC

### Goal Types & Logic
```
SAVINGS GOAL
  Fields: title, targetAmount, currentAmount, deadline
  Progress: (currentAmount / targetAmount) * 100
  Status logic:
    - on_track: progress ≥ expected_by_now (based on days elapsed)
    - at_risk:  progress < expected_by_now - 10%
    - completed: currentAmount >= targetAmount
    - failed:   deadline passed && !completed

NO-SPEND CHALLENGE
  Fields: title, category, durationDays, startDate
  Progress: days elapsed without a transaction in category
  Status: at_risk if any expense in category detected

BUDGET LIMIT
  Fields: title, category, limitAmount, month
  Progress: (spent_in_category / limitAmount) * 100
  Status: at_risk if > 80% used before month end

SAVING STREAK
  Fields: title, streakTarget (days), streakCount, lastContributionDate
  Progress: (streakCount / streakTarget) * 100
  Logic: increment streakCount when a savings transaction added same day
         reset streakCount if a day is missed
```

## RIVERPOD STATE PATTERNS

### Use this pattern for ALL async Firestore data:
```dart
@riverpod
Stream<List<TransactionEntity>> transactions(TransactionsRef ref) {
  final user = ref.watch(currentUserProvider);
  if (user == null) return const Stream.empty();
  final repo = ref.watch(transactionRepositoryProvider);
  return repo.watchTransactions(uid: user.uid);
}
```

### Use this pattern for UI consumption:
```dart
final transactionsAsync = ref.watch(transactionsProvider);

return transactionsAsync.when(
  data: (transactions) => transactions.isEmpty
      ? const FinnEmptyState(
          message: 'No transactions yet',
          ctaLabel: 'Add your first one',
        )
      : TransactionDateGroupedList(transactions: transactions),
  loading: () => const FinnShimmerList(itemCount: 6),
  error: (e, st) => FinnErrorWidget(
    message: 'Failed to load transactions',
    onRetry: () => ref.invalidate(transactionsProvider),
  ),
);
```

### Use Either for all use case return types:
```dart
// domain/usecases/add_transaction.dart
class AddTransaction {
  final TransactionRepository _repo;
  AddTransaction(this._repo);

  Future<Either<Failure, Unit>> call(TransactionEntity transaction) =>
      _repo.addTransaction(transaction);
}
```
